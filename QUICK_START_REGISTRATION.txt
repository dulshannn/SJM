================================================================================
  HOW TO USE
================================================================================

STEP 1: Start the Server
--------------------------
cd /tmp/cc-agent/61817615/project
php artisan serve

Navigate to: http://localhost:8000

STEP 2: Access the Landing Page
--------------------------------
The default route (/) now shows the enhanced landing page with:
- Hero section
- Features showcase
- Security section
- Login and Register buttons in navbar

STEP 3: Register a New User
----------------------------
Option A - Via Modal:
  1. Click "Register" button in navbar
  2. Modal opens with registration form
  3. Fill all fields
  4. Submit

Option B - Via Standalone Page:
  1. Navigate to /register
  2. Fill registration form
  3. Submit

Required Fields:
  - Full Name
  - Email (must be unique)
  - Phone Number
  - Account Type (Customer/Supplier/Delivery)
  - Password (min. 8 characters)
  - Confirm Password (must match)

STEP 4: Verify OTP
------------------
After registration:
  1. Redirected to /otp page
  2. Check OTP in logs:
     tail -f storage/logs/laravel.log
  3. Copy 6-digit OTP code
  4. Enter OTP in verification page
  5. Click "Verify OTP"
  6. Redirected to dashboard

STEP 5: Access Dashboard
-------------------------
After successful OTP verification:
  - Automatically logged in
  - Redirected to role-based dashboard
  - Full system access granted

================================================================================
  NEXT STEPS FOR PRODUCTION
================================================================================

Required:
  1. Update database password in .env:
     DB_PASSWORD=your_actual_supabase_password

  2. Install PHP extensions:
     sudo apt-get install php-pgsql php-xml php-dom php-mbstring

  3. Configure email for OTP delivery:
     MAIL_MAILER=smtp
     MAIL_HOST=your-smtp-host
     MAIL_PORT=587
     MAIL_USERNAME=your-email
     MAIL_PASSWORD=your-password

  4. Update AuthController to send OTP via email instead of logging

  5. Set production environment:
     APP_ENV=production
     APP_DEBUG=false

  6. Cache configuration:
     php artisan config:cache
     php artisan route:cache
     php artisan view:cache

Recommended:
  - Enable HTTPS/SSL
  - Add rate limiting
  - Implement CAPTCHA
  - Add email verification
  - Setup monitoring
  - Regular backups
